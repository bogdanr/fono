Key findings (read first)

  • On CPU, quantization gives essentially zero speed benefit on every laptop CPU we measured. Quant on CPU is purely a memory/disk optimization (≈30–55 % less RSS, ≈55–80 % less disk).
  • On Vulkan, quantization does help — especially for large-v3-turbo: q5_0 is 1.6–2.4× faster than fp16 because the iGPU is bandwidth-bound, not compute-bound. For tiny/base/small the win is small or negative.
  • q8_0 is the consistently-safe quant: ΔLev ≈ 0 on every model where we have both fp16 and q8_0 CPU measurements. q5_1 is risky — on base multilingual it costs +0.113 mean Levenshtein vs fp16 on i7-8550u (lev 0.298 vs 0.185), and on small.en it costs +0.072. q5_1 on English-only tiny.en is acc-neutral.
  • The base / base.en models are missing from the registry. Our data shows they are the universal comfortable sweet spot on CPU for every laptop including 2016-era Kaby Lake. This is a gap — the registry skips from tiny straight to small.
  • Tiny / tiny.en ship q5_1 only. Given quant saves no time on CPU and the accuracy delta is small (≤0.025) for tiny.en, this is defensible — but exposing fp16 as an opt-in for hosts with ≥4 GiB free RAM gives users the choice between "less RAM" and "more accurate".
  • Recommended registry change (data-driven): default base/base.en to q8_0 (not q5_1) because q5_1 has too much accuracy variance on multilingual base, while q8_0 is acc-neutral and only ~37 % larger.
Verdict thresholds: comfortable requires batch RTF ≥ 2.0 and stream RTF ≥ 1.5. borderline requires batch RTF ≥ 1.0. Anything else is unsuitable.

Column glossary
  • batch RTF: median audio-seconds transcribed per wall-second in batch mode. Higher is faster.
  • stream RTF: same in 1-second-chunk streaming mode. Below 1.0 means we can't keep up live; below 1.5 fails the comfortable threshold.
  • TTFF s: median time-to-first-frame across fixtures (latency on first partial result).
  • RSS MiB: peak resident set size — how much RAM the process actually held.
  • acc lev: mean per-fixture normalized Levenshtein distance from the reference text. Lower is better. 0.0 = perfect transcription on every fixture; >0.5 = severely degraded. Aggregated across all 10 equivalence fixtures (en + es + fr + zh + ro).
  • Δ vs fp16: accuracy delta of this quant cell vs the fp16 cell on the same host/backend. Positive = quant is worse.
  • size MiB: approximate disk size of the GGML weights file.
  • verdict: comfortable batch ≥ 2.0 and stream ≥ 1.5 · borderline batch ≥ 1.0 · unsuitable below.
