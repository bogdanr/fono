1 · Decision Heatmap — best CPU pick per (host, family, language)

Rows = hosts sorted by release year. Columns = model family × language. Each cell shows the best-scoring CPU build at AC power. ⚡ badge marks where the Vulkan backend improves the verdict for the same (host, family, language). Battery data hidden by default — toggle via the filter bar.

2 · Speed Sweep — batch RTF by model variant, faceted per host

One panel per host. Bars within a model variant show fp16 / q8_0 / q5_1 (q5_0 only for large-v3-turbo). Empty slots = not measured. Y axis is log; horizontal lines mark batch RTF = 1.0 and 2.0.

3 · Quant Speedup vs fp16 — VNNI vs avx2-fallback hosts

Ratio = (quant batch RTF) ÷ (fp16 batch RTF). A value of 1.0 means no speedup; values >1 mean the quantised model runs faster. Hosts with AVX-VNNI (Alder Lake+, Ryzen Zen3+) execute integer-quant kernels in vector units; pre-VNNI hosts fall back to scalar paths.

4 · CPU vs Vulkan — batch RTF and speedup

Paired bars per (host, model, quant) where both backends have data. Speedup = Vulkan RTF ÷ CPU RTF (>1 means Vulkan faster).

5 · Coverage Matrix — what's measured, what's missing

Rows = hosts. Columns = (backend × family × language × quant). Green ≥2 iterations, yellow = 1, grey = not measured. Use this to plan the next bench session.
