Generated 2026-05-23 13:51 UTC · 203 cells · 5 hosts. Companion to calibration3.html — that page diagnoses runs; this page decides which model Fono should ship to users with a given hardware fingerprint.

What is this page? An interactive policy view. Adjust the sliders below; the recommendation walk re-evaluates every measured host under the new gates, the scatter views show which host features predict viability, and the policy JSON section emits a versioned blob that a future crates/fono-stt/src/auto_select.rs consumes to drive runtime model selection. Replaces the static LocalTier::default_whisper_model() path in crates/fono-core/src/hwcheck.rs.

1 · Recommendation per measured host

For each measured host the policy walk picks the candidate that passes all hard gates (batch RTF, accuracy, memory, binary variant, language, power) under the largest-family, fp16-preferred, Vulkan-if-≥1.2×-CPU preference order. The trace cites which gate the next-up candidate failed, so you can see why a smaller model was chosen.

2 · Feature vs outcome scatter

Each dot = one measurement. Color distinguishes AVX-VNNI capable hosts (blue) from pre-VNNI hosts (orange). Shape: ● fp16 / ■ q8 / ▲ q5. Size: tiny → small dot, turbo → large. Dashed horizontal/vertical lines show the current threshold; dots in the upper-left quadrant of each batch-RTF panel qualify. This is where "data hides things we don't expect" becomes visible (e.g. quant uplift on Vulkan being flat across all hosts).

3 · Quant uplift per host — does quantization help here?

For each (host, build) pair, the bar shows the median quant_RTF / fp16_RTF across all measured (family, language) combinations. A categorical tag labels each bar: large (≥1.5×), moderate (1.1–1.5×), none (0.9–1.1×), regression (<0.9×). Use this to discover host classes where the policy should force fp16 (typically Vulkan builds across all measured hosts).

4 · Pareto frontier per host — what's the recommendation trading?

Each scatter = one host. X-axis is worst-fixture WER (accuracy_en_max, lower=better) so a single catastrophic transcript can't hide behind a friendly mean. Y-axis is batch RTF (higher=better). The chart caps display at WER ≤ display_wer_cap (default 0.30); cells worse than that are dropped from the plot but still counted in section 6. The Pareto frontier is the set of cells where no other cell is both more accurate AND faster. The current recommendation is marked with a target reticle (◎). Dashed lines show the current thresholds; the qualifying quadrant is upper-left.

5 · Policy JSON export

Versioned blob describing the rules currently selected by the sliders. Copy this into crates/fono-stt/src/auto_select.rs (or its designated data file). The Rust side becomes a decision tree over detected host features, not a tier classifier with a hardcoded family map. evidence_hosts tells you how many machines each rule rests on — never ship a rule with n=1 without flagging it.

6 · Data gaps under current policy

The recommendation walk only sees cells that exist in the matrix. If a candidate the policy would prefer is unmeasured, the walk silently falls through to a smaller alternative.
